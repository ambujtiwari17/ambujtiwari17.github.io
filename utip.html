Client and Problem Space

The University of Texas Inequality Project(UTIP) is a small research group housed within the LBJ School of Affairs, UT Austin that is concerned with measuring and explaining movements of inequality in wages and earnings and patterns of industrial change around the world. However, the UTIP website had several design and development issues, the three key issues being:

  • Responsiveness
  • Visualization of global research data
  • User-friendly design

It was clear that the website needed a major design overhaul.

Design Goal: How do we design a website that is fully responsive, leverages the richness of research data in the form of engaging, interactive data vizualizations and is visually appealing(professional yet neat and clean)?

Solution

The rebranded and redesigned website addresses these key issues as previously identified. It is a fully responsive website that works smoothly across multiple device screens, it uses interactive JavaScript map libraries to represent the large amount of research data and maintains the professional branding and ethos of University of Texas web design practices.

User Research

Competitive Analysis

For our competitive analysis, we decided to look at four other websites. We identified three direct competitors which operate in the same realm of global income inequality research and data representation namely – World Inequality Database(WID), Fred Solt’s SWIID and World Bank's WDI – and one indirect competitor in UT LBJ's own CHASP(Center for Health and Social Policy)

The findings from the competitor analysis are summarized as follows:

Direct

  1. World Inequality Database(WID)

    2. Strengths

    • Simple, clean interface
    • Relative income position simulator on homepage which is easy to take
    • Faceted search allows refined search results
    • Three main cards on homepage for world view, country graphs and data tables
    • Interactive charts and maps (powered by Highcharts)
    • Easy data download options available (based on indicators, regions, years)
    • Country wise inequality and macroeconomic data available
    • News items have more context involved

    Weaknesses

    • Not responsive
    • No clear instructions on how to view/access data

  2. World Bank's WDI

    3. Strengths

    • Simple, clean interface with nicely organized content
    • Fully responsive website, content scales according to the device/platform
    • Customized graphs and charts based on search criteria(countries, variables, years)
    • Detailed and interactive charts and maps (Tableau visualizations)

    Weaknesses

    • Not obvious in the first glance where to go to see income inequality related charts
    • Difficult to determine where to download data

  3. Fred Solt’s SWIID

    4. Strengths

    • Simple, clean, minimalist interface, not using many colors which gives it a serious and professional look
    • Customized graphs and charts based on search criteria (countries, variables, years)
    • Allows comparison of multiple countries data
    • Three main cards on homepage for world view, country graphs and data tables
    • Charts and maps (powered by Shiny, an R package)
    • Easy data download options available (based on indicators, regions, years)
    • Country wise inequality and macroeconomic data available
    • News items have more context involved

    Weaknesses

    • Not fully responsive, only partially
    • Charts not interactive on homepage (Source Data however, is interactive)
    • Only 4 countries allowed at a time for comparison

Indirect

  1. CHASP (Center for Health and Social Policy)
    • Fully responsive website
    • Consistent with the University of Texas brand image
    • On the homepage, information about the center is clear, communicates purpose effectively
    • Clear navigation hierarchy on the top with clear labels(navigation links/items)

Usability Evaluation

We also conducted a preliminary usability evaluation on the UTIP website based on Nielsen's 10 Heuristics for Usability Design. This allowed us to identify the key usability issues and helped us target our focus on these and build design solutions around them.

Key findings from the evaluation were:

  • Unnecessary black gutter space uses a lot of screen estate, loss of valuable space that can be used to showcase important information
  • The World Map GIF on the website does not give viewer enough time to look at the trends
  • Static GIF does not allow interactivity with the World Map and the underlying data
  • Switch global navigation links on the left with current links sitting at the top
  • Redundant links in the bottom and the top can confuse the user to some extent
  • News items can be changed, instead of just links they can be “cards”
    • Each card consists of an image (from event/presentation), title, overview and button that opens a full page for that news item/event with appropriate media
  • Inconsistent usage of typography
  • Archive section can exist separately, away from the homepage
  • Lack of data set visualizations(charts, maps, graphs)

Information Architecture

Since the website had a lot of information and data that needed to be presented effectively, information architecture of the website was pretty crucial. The redesigned websited had to have a solid yet flexible framework for file and directory structure.

The older website did not have organized folders for the different web components and media files (audio, video, PDF documents etc.). Everything was placed right in the root folder, which made the website structure clunky and made the navigation difficult. So the first step towards creating an effective website architecture was to have three main directories – img, css, js – for images, css and javascript files. Then the other files would be placed in other directories.

Content Inventory

Creating a content inventory helped us build a solid foundation for our project in terms of the levels of navigation we would aim for, the contents on each level and the overall project hierarchy.

We defined two levels of navigation for the content and the initial structure of our website. We decided to have a total of six global navigation items - Home, About, Research, Tools, Tutorials, Publications and Contact Us. The initial two items "Working Papers" and "Data Sets" were grouped under a new nav item "Research" to effectively attract the user browsing the website to the research done(being done) by the group. We also added navigation breadcrumbs in our design so that the user is aware of what level they are on the site hierarchy and can use those to navigate to derivative pages

Designing and Prototyping

Design Mockups

Once we had conducted our research and analysis, we started working on mockups for the UI screens for three primary devices – desktops and laptops, tablets and mobile phones.

Development

The development of the project took a significantly long time. We started by creating a basic HTML webpage, and then we added the bells and whistles to it. We used the Bootstrap framework for CSS styling and an extremely powerful JavaScript library called amCharts for data representation on world maps. It is an open source library and has great functions for interactive data visualizations.

The amCharts JavaScript library allowed us to present the richness of the UTIP EHII research data in a new light. Instead of just seeing static GIF images of the global Gini coefficients, the user could now view the Gini coefficients for individual countries, even the smallest ones, by zooming in and out. Even on mobile phones, the user has the ability to now pinch and zoom and interact with the charts. We streamlined the entire website structure to make it feel clean and robust. Using Bootstrap 4 ensured that we could scale the website on multiple device resolutions. The site also loaded about 30x faster than before.

We also used GitHub for version control and that was an immense help. Any time we wanted to roll back to a version, it was made possible using version control.
